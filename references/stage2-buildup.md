# 阶段二：能力补齐与实操

前提：阶段一已完成（档案里有方向、目标JD、AI水平定级、差距清单）。如果没有，回到阶段一。

本阶段的核心信念，要在开场就传递给用户：

> 转行AI岗，面试官最怕招到"只会聊AI概念的人"。打败这种怀疑的唯一办法，是你拿出一个**自己动手做出来的、解决了真实业务问题的东西**。哪怕很小。所以这个阶段不是"上课"，是"做项目"，知识是边做边补的。

## 2.1 知识补齐：只学JD要的

拿出差距清单，**严格按目标JD要求排优先级**，JD没要求的坚决不学（告诉用户："转行不是考研，大而全是焦虑陷阱，贴JD才是最短路径"）。

每个知识点给一个**最小学习闭环**：看什么（≤1小时的输入）→ 做什么（一个动手动作）→ 怎么验证学会了（能向外行讲清楚 / 能操作出结果）。例如：

- **提示词**：看一篇结构化提示词方法（角色+任务+约束+示例）→ 给自己工作里的真实任务写3条提示词并迭代 → 验证：同一任务，迭代后的输出明显可用
- **RAG/知识库**：用一句话+类比理解（"开卷考试：先去你的资料库里翻到相关页，再照着答"）→ 用IMA或NotebookLM把自己的工作文档建成知识库问答 → 验证：能回答出文档里的细节
- **Agent/工作流**：理解"会自己决定步骤并调用工具的AI"→ 在Coze上跑通一个官方模板 → 验证：能说出这个模板里每个节点在干嘛
- **模型基本原理/SFT/RL**：找2-3个高质量科普视频（B站搜"大模型原理 科普"）→ 验证：能用自己的话向完全不懂的朋友讲明白"AI为什么会胡说八道"

知识补齐和2.2/2.3的实操**并行推进**，不要让用户"学完再做"。

## 2.2 在职用户：在现有工作中找AI落地点

在职是巨大优势：有真实业务、真实数据、真实用户。引导步骤：

**第一步：扫描提效点。** 问用户："过去一个月，哪些工作让你觉得重复、机械、不想干？"典型靶子：重复回复的消息/邮件、周报月报、数据整理和报表、话术和文案的批量生产、信息搜集和摘要、审核和检查类工作。

**第二步：用MVP法则选一个切入。** 给用户讲清MVP（最小可行产品）："不要憋大招。选**一个**最痛的小问题，做一个能跑起来的最简版本，1-2周内见效。先证明能用，再考虑做好。"选择标准：①每周至少耗你2小时以上 ②流程相对固定 ③效果可以量化（省时、降错、提量）。

**第三步：按用户能力选工具。**

- **零代码**：扣子（Coze）、Dify、WorkBuddy——搭Agent和工作流，拖节点配提示词即可。国内用户首推Coze（生态全、教程多、可直接发布成机器人给同事用）
- **愿意碰一点代码**：直接和 Claude / Codex 对话式开发（vibe coding）——让AI写一个小工具网页/脚本，用户只负责描述需求和验收。强调："你不需要会写代码，你需要会描述问题和判断结果好不好——这恰恰就是AI产品经理的核心工作。"
- 选择困难时的默认建议：流程自动化类 → Coze工作流；问答/知识类 → Coze Agent + 知识库；需要个性化界面或计算 → Claude写小工具

**第四步：落地并量化。** 做出来后真的用到工作里（或给同事用），记录效果数据：节省多少小时/周、错误率降了多少、覆盖了多少单量。**这些数字就是未来简历和面试的弹药。**如果公司氛围允许，把成果汇报给上级——"在司推动AI提效"本身就是JD高频要求。

## 2.3 已离职用户：围绕过往痛点搭Demo

没有在职场景，就回到过往：

**第一步：痛点考古。** "回想上一份工作，哪个问题如果当时有个AI工具就能解决？哪个流程最浪费团队时间？"

**第二步：同样MVP。** 选一个做成Demo，要求能演示、贴行业。示例（按行业）：

- 电商运营 → 智能客服话术库Agent（输入客户问题，按品类/场景输出合规话术）；竞品评论分析工作流（爬取→分类→洞察摘要）
- 教培 → 学情分析报告生成器；家长沟通话术Agent
- HR → JD自动生成与简历初筛工作流
- 内容运营 → 选题-脚本-分发的一条龙工作流

**第三步：用模拟数据跑通。** 没有真实数据就自造或找公开数据，重点是流程完整、能现场演示、能讲清"如果接入真实业务，预期解决什么、怎么衡量"。

## 2.4 项目STAR文档（本阶段产出）

每个做完的项目，立刻和用户一起沉淀一份STAR文档（趁热打铁，细节最全）：

- **S 背景**：什么业务、什么痛点、痛到什么程度（有数字）
- **T 任务**：你给自己定的目标是什么
- **A 行动**：技术选型（为什么用Coze不用Dify？为什么这样设计流程？）、搭建过程、踩了什么坑怎么解决的、迭代了几版为什么
- **R 结果**：量化效果 + 学到了什么 + 如果重做会怎么改进

特别提醒用户记录**踩坑和取舍**——面试官追问的全是这些（"为什么这么设计？""遇到效果不好怎么办的？"），答得出细节才证明是真做的。

## 阶段二收尾

完成标志：1-3个可演示项目 + STAR文档，写入转行档案。和用户确认：能不能对着Demo流畅地讲5分钟？可以的话，进阶段三。
