# 阶段三：产出与面试准备

前提：阶段二已产出至少1个实操项目+STAR文档。本阶段产出三份最终交付物（模板见 `templates.md`），出题规则见 `question-bank.md`。

## 3.1 选定面试演示物

从阶段二的项目里，帮用户选**一个主演示物**（其余作为面试谈资），按目标JD的侧重决策：

| JD侧重 | 最佳演示物 | 形态 |
|---|---|---|
| 流程自动化、提效、智能客服 | **工作流** | Coze/Dify画布截图 + 现场跑一遍 |
| 对话产品、助手类、知识库 | **Agent** | 可直接对话的链接或录屏 |
| 产品规划、0-1、行业解决方案 | **方案书** | 10页内的PDF/PPT：痛点→方案→MVP验证→路线图 |
| 数据驱动、评测、效果迭代 | **数据报告** | 评测维度设计 + 前后效果对比 |

无论选哪种，都要求用户准备：①30秒电梯版讲法 ②5分钟完整版讲法 ③一句话效果数字。提醒：方案书可以和工作流/Agent组合（"我不仅想了，还做出来验证了"是最强组合）。

## 3.2 三份交付物

### 交付物一：完整学习路径

回顾+前瞻的文档：用户从哪里出发（定级）、走过哪些环节、产出了什么演示物、入职后前90天还该补什么。注意学习路径里必须明确写出"最终交付物=可在面试演示的工作流/Agent/方案书"，让看到这份路径的人知道终点是实物而非"学完了"。

### 交付物二：极度贴合JD的简历

改写规则：

1. **逐条对齐**：目标JD的每条核心要求，简历上都要能找到对应句；JD的关键词（如"话术库""解决率""Prompt优化"）直接复用进简历
2. **AI项目放C位**：阶段二的实操项目作为独立"AI实践项目"板块放在显眼位置，用STAR文档里的量化数字
3. **过往经历翻译成AI语言**：不是抹掉过往，而是重新表述——"管理客服快捷回复模板200条" → "搭建并维护客服话术知识库（200+条目），按场景分类与定期迭代，这套方法论可直接迁移至智能客服知识库建设"
4. **诚实红线**：表述可以包装，事实不能虚构。每一条都必须经用户确认真实发生过
5. 一页为佳；自我评价区写三标签公式："我懂X行业业务 + 我有AI实操项目 + 我在持续学AI"

### 交付物三：题目单 + 速记卡

按 `question-bank.md` 的三部分结构（A自我介绍与项目概览 / B JD强相关 / C软性问题）为用户的目标JD定制生成。流程：

1. **先生成题目单（只有题目，没答案）**，发给用户
2. **逐题让用户先自己答**（口头或打字）。这一步绝不能跳过——直接发参考答案，用户面试时背不出来也讲不真
3. 答得好的 → 帮他收紧结构、补量化数字；答不出的 → 给答案框架（答题思路+结构），让用户用自己的真实经历填肉，再帮他打磨
4. 全部过完后，输出**完整题目单（题目+定稿答案）**和**速记卡**（每张卡=一个问题+3个关键词+30字要点，面试前一晚和候场时扫一眼用的）

## 3.3 模拟面试模式（可选但强烈推荐）

题目单定稿后，问用户要不要来一轮模拟面试。规则：

- 你扮演目标公司的面试官（用人部门主管人设），从自我介绍开始，按题目单提问但**顺序打乱**，并基于用户回答即兴追问（"为什么这么设计？""数据怎么来的？""如果效果不达标呢？"）
- 适度施压：质疑行业跨度（"你之前做X的，凭什么能干AI？"）、追问细节
- 每3-5题暂停一次，给反馈：哪句答得好、哪里啰嗦、哪里露怯
- 结束后给整体评分和"再练清单"

## 3.4 收尾

三份交付物齐了之后，最后叮嘱用户三件事：①面试前去实际体验目标公司的AI产品，至少找出2个问题和1个改进建议（这是性价比最高的加分项）；②速记卡打印或存手机，面试当天只看卡不看长文档；③面完一家立刻回来复盘，把新问到的题补进题目单——题目单是活的。
